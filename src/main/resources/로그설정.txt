

1. log4jdbc.log4j2를 사용하고 있으면 log4jdbc.log4j2.properties 파일을 만들어야 함.
	- 그래야 log4j2.xml 파일을 정상적으로 읽을 수 있음.
	
2. 레벨 설정은 application.properties 파일에서 하기

# log4jdbc의 모든 상세 로그를 OFF 합니다.
logging.level.log4jdbc.log4j2=OFF
logging.level.jdbc.audit=OFF
logging.level.jdbc.resultset=OFF
logging.level.jdbc.connection=OFF
logging.level.jdbc.sqlonly=OFF

# 오직 쿼리 실행 시간만 출력하는 이 로거만 DEBUG로 켭니다.
logging.level.jdbc.sqltiming=OFF


3. 나머지 모든 logger의 level이 off인 경우
logging.level.log4jdbc.log4j2=DEBUG 만 설정한다고 해서 로그가 나오는게 아님.
logging.level.log4jdbc.log4j2=DEBUG 로 설정하고 보고 싶은 로거의 레벨을 추가로 설정해줘야 함.

예를 들어 쿼리와 쿼리 시간을 보려면 아래처럼 2개 로거를 켜야 함.
logging.level.log4jdbc.log4j2=DEBUG
logging.level.jdbc.sqltiming=DEBUG

resultset을 보고 싶으면 아래처럼 2개 로거를 켜야 함
logging.level.log4jdbc.log4j2=DEBUG
logging.level.jdbc.resultset=DEBUG
